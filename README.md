# Квиз Плиз MVP

Веб-приложение для создания и проведения квизов в реальном времени.

## Возможности

- регистрация и авторизация участников и организаторов;
- создание, редактирование и удаление квизов;
- категории, время и вес каждого вопроса;
- текстовые вопросы и вопросы с изображением;
- одиночный и множественный выбор ответа;
- подключение к активной игре по шестизначному коду;
- синхронный показ вопросов через WebSocket;
- экран правильных ответов и промежуточный лидерборд;
- итоговый результат и сохранение истории игр;
- интерфейс для компьютеров и ноутбуков.

## Запуск

Требуется Node.js 20 или новее. Внешние пакеты не используются.

```bash
npm start
```

Если команда `npm` недоступна, сервер можно запустить напрямую:

```bash
node server/native.js
```

После запуска приложение доступно по адресу [http://127.0.0.1:3001](http://127.0.0.1:3001).

Для демонстрации откройте приложение в двух разных браузерах или в обычном и приватном окне:

1. Зарегистрируйте организатора и создайте квиз.
2. Нажмите «Запустить» и скопируйте код комнаты.
3. Во втором окне зарегистрируйте участника и введите код.
4. Запустите игру, ответьте на вопросы и завершите квиз.

## Проверка

```bash
npm run build
```

Эквивалент без `npm`:

```bash
node --check server/native.js
node --check src/app.js
```

Команда проверяет синтаксис клиентской и серверной частей. Основные сценарии проверяются вручную в двух браузерных окнах: одно для организатора, второе для участника.

## Хранение данных

Пользователи, квизы и игровые сессии сохраняются в `server/data.json`. Файл создаётся автоматически и исключён из Git, чтобы тестовые аккаунты и результаты не попадали в репозиторий.

Пароли хешируются с помощью `scrypt`. Токены подписываются HMAC-SHA256. Для размещения приложения в интернете необходимо задать собственный секрет:

```bash
AUTH_SECRET="replace-with-a-long-random-value" npm start
```

## Структура

- `index.html` — точка входа;
- `src/app.js` — клиентское SPA, маршруты и WebSocket-клиент;
- `src/styles.css` — стили интерфейса;
- `server/native.js` — HTTP API, WebSocket-сервер и игровая логика;
- `server/native-auth.js` — хеширование паролей и токены;
- `server/store.js` — файловое хранилище;
- `server/quiz-validation.js` — проверка структуры и настроек квиза;
- `server/scoring.js` — проверка ответов и подсчёт баллов;
- `outputs/architecture.md` — архитектура и соответствие требованиям.
